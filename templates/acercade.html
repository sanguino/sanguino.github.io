Defensa del proyecto

Introducción

He decidido utilizar Angular debido a que se quiere hacer una SPA y que se quiere ampliar a futuro. La aplicación debe usar un framework MVC moderno, flexible y escalable como Angular, aunque se podrían haber elegido muchos otros. En concreto la parte de conexión producto - carrito se simplifica mucho con el databinding de Angular. Además es muy fácilmente testable, y aunque no se han introducido tests de momento, a futuro nos ayudará a mantener la aplicación con seguridad. En un principio pensé en usar Angular 2, que acaba de pasar de alpha a beta, pero no tengo aún la soltura que me gustaría.

Respecto al diseño, he decidido usar Bootstrap para el grid y estructura general. No he usado muchos elementos propios, siempre que he podido he usado componentes de Bootstrap para poder centrarme más en el responsive y la experiencia de usuario.

Debido a que se van a querer realizar estudios sobre los hábitos de compras de los clientes, optaría por utilizar Google Analytics trazando eventos para cada una de las acciones. No he implementado esta solución, pero es la que utilizaría por el tipo de proyecto y teniendo en cuenta que es una SPA.

Para la conexión a BBDD  propongo el uso de una API REST en JSON, realizada con Node.js, Express, MongoDB. Para este ejemplo se ha realizado un "fake" de conexión con archivos .json. Si se tuvieran los servicios, solo con cambiar la ruta de los .json a la ruta de los servicios valdría.

Estructura general

Se ha optado por el uso de Gulp para automatizar tareas. Se usa Browserify para importar módulos, Sass como compilador CSS, Connect para levantar un server de desarrollo y un par de watchers para los .js y .sass.

En la carpeta app se encuentra la aplicación, el router, los controladores y servicios que se importan con Browserify.

En la carpeta sass se encuentra un fichero scss que se compila en main.css dentro de public.

El resto de ficheros están directamente en la carpeta public.

Si se desea ejecutar en local, descargar las fuentes y ejecutar npm install para instalar los modulos y gulp default para precompilar y lanzar el server, que estará disponible en http://localhost:4000. Es posible que para poder usar el precompilador de sass se necesite ejecutar [sudo] gem install sass

Historia 1: el usuario quiere ver los productos disponibles agrupados por categorías

Se leen los datos de la API a través del servicio de productos, y se agrupan por categorías. Por cada producto se crea un bloque con su información y se muestran ordenados por precio ascendente.

Historia 2: el usuario quiere añadir productos a su carrito y ver el importe total de la compra

Los productos se añaden por número de unidades al carrito. Se crea en la colección del carrito un objeto con el número de unidades y el producto asociado. Si se añaden más elementos se suman automáticamente.

Historia 3:  el usuario quiere tramitar un pedido

Al pinchar en el checkout ,se previsualíza la compra y se pide el usuario. Si coincide con alguno de los que se leen con el servicio de usuarios, se pasa al resumen de compra. Si no coincide, se muestra un alert sugiriendo el primer usuario leído de la API (para comodidad de testing). En el resumen de compra se muestran todos los datos solicitados. En caso de cargar la URL del resumen de compra sin haber validado el usuario, la app redirige al listado de productos automáticamente.
