Defensa del proyecto

Introducción

Debido a que la se quiere hacer una SPA y que se quiere ampliar a futuro la aplicacion se debe usar un framework mvc moderno. He decidido utilizar angular por varias razones, por la flexibilidad, facilidad de desarrollo, etc. Pero en concreto la parte de producto - carrito se simplifica mucho con el databinding de angular. Ademas es muy facilmente testable, y aunque no se han introducido test de momento, a futuro nos ayudara a mantener la aplicación con seguridad.

En un principio pense en usar angular 2, que acaba de pasar de alpha a beta, pero no tengo aun la soltura que me gustaría y como os comente en su momento, estos dias salgo a las tantas, sumado a las cenas de navidad, compras... en resumen solo tenia parte del fin de semana para hacerlo.

Respecto al diseño he decidido usar bootstrap para el grid y estructura general. No he usado apenas elementos propios, he usado muchos componentes propios de bootstrap, para poder centrarme más en el responsive y la experiencia de usuario.

Debido a que se van a querer realizar estudios sobre los habitos de compras de los clientes, la idea es utilizar google analytics, trazando eventos para cada una de las acciones. No he implementado esta solución, pero es la que utilizaría por el tipo de proyecto y teniendo en cuenta que es una SPA.

Para la conexión a BBDD se propone el uso de una API REST en JSON. La idea sería realizarla con node, express y mongodb. Para esté ejemplo se ha realizado un "fake" de conexión con archivos .json. Si se tuvieran los servicios, solo con eliminar .json de la ruta de los servicios valdría.

Estructura general

Se ha optado por el uso de gulp para automatizar tareas. Se usa browserify para importar modulos, sass como compilador css, connect para levantar un server de desarrollo y watch para que se ejecuten las tareas automaticamente.

En la carpeta app se encuentra la app, el router, los controladores y servicios que se importan con browserify.

En la carpeta sass se encuentra un fichero scss que se compila en main.css dentro de public

El resto de fichero estan directamente en la carpeta public
